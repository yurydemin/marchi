# Changelog

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), версии — на
[Semantic Versioning](https://semver.org/lang/ru/).

## [0.1.0] — 2026-07-23

Первый релиз. Semver 0.x — сознательный выбор: проект ещё не проходил security-аудит
третьей стороной и не имеет внешних пользователей, поэтому 0.x честнее сигнализирует
«API и формат данных могут измениться между минорными версиями», чем сразу 1.0.0.

### Добавлено

**Основной движок (архивация)**
- IMAP-аккаунты: пароль или OAuth2 (Google/Microsoft, BYO-приложение).
- Инкрементальная синхронизация по UID с устойчивостью к обрыву (проверено принудительным
  убийством процесса в разных точках синхронизации — ни дублей, ни потери писем).
- Хранение в формате Maildir (`tmp/`→`new/`→`cur/`), атомарная запись только после
  успешного коммита метаданных в SQLite.
- Извлечение вложений и MIME-разбор (`emersion/go-message`).
- Master Key (Argon2id) поверх отдельного Data Encryption Key — смена пароля не требует
  перешифровывания уже сохранённых данных.

**Правила архивации**
- Визуальный AND/OR-конструктор условий, 15 типов условий, вложенность до 3 уровней.
- Действия: архивировать / пропустить / архивировать и удалить с сервера / архивировать
  и отметить прочитанным.
- Приоритет правил с drag-and-drop переупорядочиванием.
- Опциональный `rules.yaml` — однонаправленная синхронизация файла в БД через fsnotify.

**Поиск**
- Полнотекстовый индекс (Bluge) по теме, телу, отправителю, вложениям.
- Фильтры по дате, аккаунту, папке, наличию вложений; сортировка по релевантности или
  дате.
- Пересборка индекса из архива без обращения к сети (Web UI и `marchi reindex`).

**S3-репликация**
- Зеркалирование заархивированных писем в любое S3-совместимое хранилище.
- Клиентское шифрование (AES-256-GCM) перед загрузкой, проверка целостности по SHA-256.
- Ленивая подгрузка S3-резидентных писем с байт-бюджетным LRU-кэшем.
- Retention: трёхстадийный жизненный цикл (локально → S3 → удаление) с глобальными
  дефолтами и переопределением на уровне аккаунта.

**Восстановление**
- Восстановление одного письма или пачки (по результатам поиска) в любой IMAP-аккаунт
  через `APPEND`, с автоматическим фолбэком на SMTP.
- История восстановлений по каждому письму.
- Массовый экспорт выбранных писем (или результатов поиска) в `.zip`.

**Web UI**
- Dashboard, Accounts, Archive (поиск + просмотр + восстановление), Rules, Settings.
- Тёмная/светлая тема (по умолчанию — как в системе, с ручным переключателем).
- Локализация RU/EN — интерфейс, серверные сообщения обратной связи, CLI `--help`.
- WebSocket-прогресс для длительных операций (синхронизация, восстановление, экспорт,
  переиндексация).

**CLI**
- `add-account`, `list-accounts`, `test-connection`, `sync`, `status`, `logs`,
  `retention run`, `reindex`, `unlock`, `config show`, `migrate`.

**Эксплуатация**
- Systemd unit с hardening (`ProtectSystem=strict`, некорневой пользователь) —
  подтверждено живьём против реального systemd.
- Docker-образ на базе distroless (non-root, ~45 МБ) + `docker-compose.yml` с опциональным
  MinIO-профилем для локального теста S3.
- Логирование одновременно в файл (`{data_dir}/logs`, ротация NFR-RL-04) и в stdout по
  умолчанию (`app.log_output`) — `docker logs`/`journalctl -u marchi` показывают логи без
  дополнительной настройки, `marchi logs` продолжает работать поверх файла.
- Prometheus-метрики (`/metrics`): письма, аккаунты, синхронизации, очередь S3,
  HTTP-latency.
- Релизная автоматизация (goreleaser) — бинарники `linux/amd64` и `linux/arm64`.
- CI на GitHub Actions: `gofmt`/`vet`/`test` (включая Docker-интеграционные тесты против
  реальных Dovecot/MinIO), `govulncheck`, проверка пересборки CSS.

### Безопасность

- Все секреты (IMAP-пароли, OAuth2-токены, S3-ключи) хранятся только в зашифрованном
  виде, ни разу не попадают в логи.
- CSRF-защита на всех небезопасных методах, XSS-санитизация HTML писем (allowlist,
  без `<img>`/`data:`).
- `govulncheck` в CI на каждый push/PR.
- Аудит лицензий всех 89 зависимостей — MIT/Apache-2.0/BSD/MPL-2.0, копилефт-лицензий нет.
