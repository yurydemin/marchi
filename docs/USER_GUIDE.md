## Первая разблокировка

При первом открытии Web UI вместо привычного интерфейса появится экран **«Разблокировка
Marchi»** с одним полем — паролем. Это Master Key: единственный секрет, которым
защищено всё остальное (пароли IMAP-аккаунтов, OAuth2-токены, ключи S3). Введите пароль
от 12 символов — он будет задан как новый.

При каждом следующем запуске процесса (перезапуск контейнера, `systemctl restart`,
обновление) сервис снова стартует заблокированным, и тот же пароль его разблокирует.
Это не баг: пока сервис заблокирован, фоновая синхронизация не запускается — так короче
окно, в течение которого расшифрованные данные вообще существуют в памяти процесса.

Забытый пароль восстановить нельзя — это не «сброс через email», а ключ шифрования.
Единственный выход — начать с чистого `data_dir` (все ранее заархивированные письма при
этом останутся нечитаемыми, если не было отдельной резервной копии).

## Добавление почтового аккаунта

Раздел **Аккаунты** → форма «Добавить аккаунт»:

- **Email** и **IMAP-сервер** обязательны.
- **IMAP-порт**: можно оставить пустым — по умолчанию 993 для SSL/TLS, 143 для
  остальных режимов.
- **TLS**: `SSL/TLS` (implicit TLS, порт 993 по умолчанию) — стандартный выбор для
  большинства провайдеров; `STARTTLS` — для серверов, которые сначала принимают
  соединение в открытом виде и потом повышают его до TLS; «Без шифрования» — только
  для локальных тестовых серверов, никогда для реального почтового провайдера.
- **IMAP-логин**: если отличается от email (некоторые провайдеры это допускают).
- **IMAP-пароль**: для Gmail/Outlook с включённой двухфакторной аутентификацией — это
  почти всегда **пароль приложения** (app password), не основной пароль аккаунта; для
  них также доступна OAuth2-авторизация — см. [Настройки → OAuth2](#настройки).

После добавления нажмите **Проверить** в строке аккаунта — Marchi подключится и
покажет либо число найденных папок, либо текст ошибки сервера (неверный пароль,
недоступный хост, отказ TLS — сообщение от сервера показывается как есть).

Синхронизация запускается автоматически по расписанию (`sync.default_schedule` в
конфиге, по умолчанию — раз в 6 часов), а также вручную — кнопка **Синхронизировать**
на Dashboard рядом с аккаунтом.

## Dashboard

Сводка по всему архиву: сколько писем всего, сколько аккаунтов активно, сколько места
занято локально и в S3, состояние очереди загрузки в S3. Ниже — таблица аккаунтов с
последним статусом синхронизации по каждому.

Пока S3 не настроен, соответствующие плитки показывают «Не настроено» — это ожидаемое
состояние, не ошибка.

## Правила архивации

Раздел **Правила**. По умолчанию, если правил нет — архивируется всё. Правило состоит
из:

- **Условий** — дерево из групп «ВСЕМ (И)» / «ЛЮБОМУ (ИЛИ)», вложенность до 3 уровней.
  Каждое условие — конкретный тип (домен отправителя, тема по regex, размер, дата,
  наличие вложений и т.д. — полный список в выпадающем списке добавления условия) плюс
  значение.
- **Действия**: `Архивировать`, `Пропустить` (не архивировать вовсе), `Архивировать и
  удалить с сервера`, `Архивировать и отметить прочитанным`.
- **Приоритета** — правила проверяются сверху вниз, побеждает первое совпавшее. Порядок
  меняется перетаскиванием строки за маркер `⛿` слева.

Правила можно вести и в виде файла — `rules.yaml` в `data_dir`, если он существует,
Marchi следит за изменениями и однонаправленно синхронизирует его содержимое в БД
(изменения через Web UI обратно в файл не пишутся — редактируйте что-то одно).

## Архив: поиск, просмотр, скачивание

Раздел **Архив**. Слева — дерево «аккаунт → папки» для фильтрации по конкретному
источнику; сверху — форма поиска (текст, диапазон дат, наличие вложений, сортировка по
релевантности или дате).

В результатах у писем, физически хранящихся в S3 (а не локально), — бейдж **S3**. Клик
по теме письма открывает просмотр: заголовки, тело (HTML — санитизировано, безопасно
для просмотра; если HTML недоступен — текстовая версия), вложения отдельными ссылками,
кнопка «Скачать .eml» (оригинальный файл письма байт-в-байт). Если письмо хранится
только в S3, тело подгружается на лету при открытии — незаметно для пользователя, но
может занять секунду-две при первом просмотре после долгого хранения в S3.

## Восстановление писем

В том же разделе **Архив**, над результатами поиска — чекбоксы у каждого письма плюс
выбор целевого аккаунта и папки. Отметьте нужные письма, выберите, куда восстанавливать,
нажмите **Восстановить выбранное**. Прогресс — в реальном времени рядом с кнопкой.

Технически восстановление — это IMAP `APPEND` в целевую папку; если по какой-то причине
это невозможно (сервер не поддерживает нужную команду и т.п.), автоматически
используется SMTP-отправка как резервный способ. История восстановлений для каждого
письма видна в его собственном окне просмотра — метод (`imap_append`/`smtp`), статус,
дата.

## Настройки

Раздел **Настройки**:

- **Master Key** — смена пароля. Ничего заново не шифруется: меняется только обёртка
  ключа шифрования на диске, все уже сохранённые пароли/токены/письма в S3 остаются
  читаемыми новым паролем без каких-либо дополнительных действий.
- **Хранилище S3** — эндпоинт, регион, бакет, ключи доступа. Path-style адресация нужна
  почти всегда для MinIO и self-hosted S3-совместимых хранилищ (не для настоящего AWS
  S3). «Проверить подключение» делает безопасную read-only проверку доступности бакета,
  ничего не создавая и не загружая.
- **Приложения OAuth2 (свои)** — Marchi не поставляет общий OAuth2-клиент: нужно
  зарегистрировать своё приложение в Google Cloud Console / Azure Portal и вписать сюда
  `Client ID`/`Client secret`/`Redirect URL`.
- **Retention по умолчанию** — три числа в днях: сколько письмо живёт локально до
  переноса в S3, через сколько дней после архивации оно физически переносится в S3, и
  через сколько дней после этого удаляется окончательно (из S3 и из БД). Пустое поле —
  «никогда». Действует для всех аккаунтов, у которых нет собственного переопределения;
  сами retention-проходы запускаются ежедневно автоматически (а также вручную —
  `marchi retention run`).
- **Поисковый индекс** — «Пересобрать индекс» полностью переиндексирует архив с нуля из
  уже сохранённых `.eml`-файлов и метаданных в БД. Нужно редко: если индекс подозревается
  в рассинхроне (например, после восстановления из бэкапа без индекса) или после ручного
  вмешательства в `maildir/`.
- **Последние синхронизации** — журнал последних прогонов по всем аккаунтам: сколько
  писем обработано/заархивировано, сколько ошибок, статус.

## Тема и язык интерфейса

Кнопка в правом верхнем углу переключает тёмную/светлую тему (по умолчанию — как в
системных настройках браузера), выбор запоминается в cookie. Рядом — переключатель
`EN`/`RU`, тоже переживает перезагрузку страницы и не требует повторного входа.

## CLI-эквиваленты

Всё, что можно сделать в Web UI, частично доступно и без браузера — полезно для
скриптов и unattended-сценариев:

```bash
marchi add-account user@example.com --host imap.example.com --tls ssl
marchi list-accounts
marchi test-connection user@example.com --host imap.example.com
marchi sync user@example.com
marchi status                    # история синхронизаций по всем аккаунтам
marchi retention run             # один проход retention прямо сейчас
marchi reindex                   # пересборка поискового индекса
marchi config show               # итоговая конфигурация (дефолты + файл + окружение)
```

`marchi --help` и `marchi <команда> --help` уважают `--lang ru` (или переменную
окружения `LANG`/`LC_ALL`) — описания команд переключаются на русский так же, как и
Web UI.
